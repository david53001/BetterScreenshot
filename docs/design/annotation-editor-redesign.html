03 — PRINCIPLES

Why it looks like this

01

Native, not webby

SF Pro, vibrancy/blur materials, traffic-light chrome, SF-Symbol-style icons. It should feel like it shipped with macOS — the “right” aesthetic for a CleanShot-class tool is Apple’s, not a design trend.

02

Progressive disclosure

Eleven tools, but you never see eleven tools’ worth of options. The inspector shows only what the active tool needs. That’s the “little bit of UI that serves purpose.”

03

Icons over words

The current toolbar reads Select Arrow Line Rect Fill Oval…. Glyphs are faster to scan, language-neutral, and let the toolbar shrink into a tidy floating pill.

04

Export where the eye lands

Copy is the accent button, bottom-right, with its real shortcut on it. Save is secondary; Done is quiet. The hierarchy matches how the tool is actually used.

Honest build notes

  • Maps cleanly onto today’s code. The floating pill + inspector + action bar slot into the existing EditorWindowController layout (top toolbar → canvas → bottom actions). Tool buttons become icon NSButtons; the NSColorWell becomes the swatch row (it can still open the system picker).
  • Two things shown here aren’t in the model yet. Undo/Redo needs a document history stack, and Duplicate needs a copy op — both are small additions, flagged so the design doesn’t overpromise.
  • Curved arrows, text-style presets, crop aspect ratios are in the CleanShot spec (§3) but beyond current v1 tools — drawn here as the natural next step, not as existing features.
  • Everything else is shippable now against the 11 implemented EditorTool cases.
